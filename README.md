## 🎯 Описание

**Notes** — это современное приложение для работы с заметками, построенное на принципах объектно-ориентированного программирования (ООП) с минимальным количеством зависимостей.

### Ключевые возможности

| Функция | Описание |
|---------|----------|
| 📄 **Создание заметок** | Быстрое создание с заголовком, текстом и вложениями |
| ✏️ **Редактирование** | Изменение любого поля заметки с валидацией данных |
| 🗑️ **Удаление** | Безопасное удаление с подтверждением |
| 🔎 **Поиск** | Мгновенный поиск по заголовкам |
| 📋 **Просмотр** | Красивый вывод всех заметок с детализацией |
| 🛡️ **Валидация** | Проверка данных по бизнес-правилам |
| 📊 **Логирование** | Полный журнал всех операций |

## 🚀 Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/N-888/Notes.git
cd Notes
```

### 2. Создайте виртуальное окружение

```bash
python -m venv venv
```

### 3. Активируйте окружение

**Windows:**
```bash
venv\Scripts\activate
```

**Mac/Linux:**
```bash
source venv/bin/activate
```

### 4. Запустите приложение

**Основное приложение (ООП):**
```bash
python main.py
```

**Простой скрипт (всё в одном файле):**
```bash
python notes.py
```

**Мини-приложение (main + database + utils):**
```bash
cd mini_app
python main.py
```

**На Windows — дважды кликните:**
- `Запустить_заметки.bat` — простой скрипт
- `Запустить_мини-приложение.bat` — мини-приложение

## 🏗 Архитектура

Проект построен по принципу **разделения ответственности** — каждый модуль отвечает за свою задачу:

```
Notes/
│
├── main.py                 # 🎯 Точка входа в приложение
│
├── src/                    # 📦 Исходный код
│   ├── __init__.py         # Пакет модулей
│   ├── logger.py           # 📊 Конфигурация логирования
│   ├── models.py           # 🧱 Модель данных (Note)
│   ├── fs_driver.py        # 💾 Драйвер файловой системы
│   ├── business_logic.py   # 🧠 Бизнес-логика и валидация
│   ├── ui.py               # 🖥️ Пользовательский интерфейс
│   └── extensions.py       # 🔌 Заготовка расширений
│
├── data/                   # 📂 Папка для хранения данных
├── logs/                   # 📋 Папка для логов
│
├── .gitignore              # 🔒 Игнорируемые файлы
├── requirements.txt        # 📋 Зависимости проекта
└── README.md               # 📖 Документация
```

### Диаграмма зависимостей модулей

```

       │
```

## 🧱 ООП-принципы

### Инкапсуляция
Каждый класс скрывает свои внутренние детали и предоставляет чёткий интерфейс:

```python
# Пользователь работает только с публичными методами
logic.create_note("Моя заметка", "Текст заметки")
logic.list_notes()
logic.delete_note(note_id)
```

### Модульность
Каждый модуль можно заменить или расширить независимо:

```python
# Замена UI (например, на веб-интерфейс) не затрагивает бизнес-логику
new_ui = WebUI(logic)  # Вместо UI(logic)
new_ui.run()
```

### Расширяемость
Модуль `extensions.py` готов к подключению внешних сервисов:

```python
# Подключение Telegram-бота
extensions.start_telegram_bot("YOUR_BOT_TOKEN")

# Подключение веб-API
extensions.start_web_api(port=8080)
```

## 📊 Логирование

Все действия приложения записываются в файл `logs/notes.log`:

```
2026-06-18 12:00:00 | INFO     | notes_app | Приложение запущено
2026-06-18 12:00:05 | INFO     | business_logic | Создана заметка: 'Идеи'
2026-06-18 12:01:00 | WARNING  | fs_driver | Файл не найден: abc123.json
2026-06-18 12:02:00 | ERROR    | business_logic | Ошибка: заголовок пустой
```

### Уровни логирования

| Уровень | Описание | Пример |
|---------|----------|--------|
| `DEBUG` | Детальная отладочная информация | Путь к файлу, результат валидации |
| `INFO` | Основные события приложения | Создание, сохранение, загрузка |
| `WARNING` | Предупреждения | Несуществующий файл, пустые вложения |
| `ERROR` | Ошибки | Невалидные данные, сбой чтения |
| `CRITICAL` | Критические сбои | Файл повреждён, нет папки данных |

## 🛡️ Бизнес-правила

| Правило | Описание | Реализация |
|---------|----------|------------|
| **Title обязателен** | Заголовок не может быть пустым | `ValueError` при пустом title |
| **Attachments — список** | Вложения хранятся как список строк | `TypeError` при неверном типе |
| **Уникальность ID** | Каждая заметка имеет уникальный UUID | Автоматическая генерация |
| **Валидация данных** | Все входные данные проверяются | Методы `_validate_*` |

## 🔌 Будущие расширения

Текущая версия приложения выполняет свои задачи, но есть большой потенциал для масштабирования:

### Telegram-бот
```python
# Управление заметками через Telegram
@bot.message_handler(commands=['notes'])
def show_notes(message):
    notes = logic.list_notes()
    # ... отправка списка заметок
```

### Веб-API (Flask)
```python
# REST API для доступа к заметкам из браузера
@app.route('/api/notes', methods=['GET'])
def get_notes():
    return jsonify([n.to_dict() for n in logic.list_notes()])
```
